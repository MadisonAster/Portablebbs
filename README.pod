=head1 NAME

Portable BBS

=head2 DESCRIPTION

C<Portable BBS> is simple web application example which have following features.

=over 2

=item * Web framework is Mojolicious

=item * Database is SQLite

=item * O/R mapper is DBIx::Custom

=item * Easy setup system by Test::ModuleVersion

=back

=head1 What you can learn from this application?

=over 2

=item * C<Mojolicious> web frame work. routing, template, and controller

=item * Database access using C<SQLite> and C<DBIx::Custom>

=item * Form data validation using C<Validator::Custom>

=item * How to use C<Test::ModuleVersion> to create portable web application

=back

=head1 Setup

Install some moudles.

  perl cpanm -l extlib Module::CoreList
  perl -Iextlib/lib/perl5 t/module.t list --fail | perl cpanm -L extlib

If you don't install all module try force install.

  perl -Iextlib/lib/perl5 t/module.t list --fail | perl cpanm -n -L extlib

=head1 Test

Run test to check setup is ok.

  prove t

If all test is passed, setup is ok.

=head1 Application start and stop

=head2 C<Start>

  ./hypnotoad

Application start in backend. Port number is C<10000> by default.
See L<Mojolicious> hypnotoad command about options.

You can also restart application by same command.

=head2 C<Stop>

  ./hypnotoad --stop

You can stop application by C<--stop> option.

=head2 Start in development

  ./morbo

Application is started in C<development> mode. port is C<3000> by default.

=head1 Windows support

=head2 Start

  set MOJO_MODE=production
  perl script/portablebbs daemon --listen=http://*:10000

If you want to run background,
you can register this application as windows service.

=cut
